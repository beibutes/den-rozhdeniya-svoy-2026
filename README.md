# Мой день рождения — приглашение

Одностраничный сайт-приглашение на день рождения (41 год).
**Когда:** 9 октября 2026, 18:00 · **Где:** ресторан Svoy, ул. Жумабаева, 24, Астана.

🌐 **Живой сайт:** https://beibutes.github.io/den-rozhdeniya-svoy-2026/

## Описание
Страница-приглашение для внешних гостей: информация о событии, живой таймер
обратного отсчёта до начала и интерактивный wish-list с возможностью брони
подарков. Забронированную позицию другие гости выбрать уже не могут.

## Возможности
- **Hero-приглашение** — дата, время, адрес, возраст, фото именинника.
- **Таймер обратного отсчёта** до 9 октября 2026, 18:00 (часовой пояс UTC+5).
- **Wish-list** — 10 позиций с фото и ценовым диапазоном в тенге (₸).
- **Бронирование подарков** — гость указывает имя; занятая позиция помечается
  «✓ Уже забронировано» и блокируется для других. Имя дарителя на главной НЕ
  показывается, фото забронированной позиции становится чёрно-белым.
- **Снятие брони только автором** — гость видит «✓ Ваша бронь» и кнопку снять
  лишь на своих позициях (привязка по visitor_id, проверка на сервере).
- **RSVP «Я приду»** — подтверждение участия с именем и возможностью отменить.
- **Общая бронь в реальном времени** через Supabase (видна всем гостям сразу).
- **Панель администратора** (`admin.html`) — кто что забронировал и когда,
  число уникальных гостей и журнал посещений. Вход через Supabase Auth.
- **Адаптивная вёрстка** — корректно на телефоне и десктопе.
- Фото с автоматическим SVG-fallback, если изображение не загрузилось.

## Бэкенд и безопасность (Supabase)
Бронь и посещения хранятся в Supabase (общие для всех гостей, realtime).
Модель доступа (RLS):
- **Публике** виден только факт брони (`item_id`) и можно забронировать
  (через функцию `reserve_item`). Имена дарителей и журнал визитов прочитать
  нельзя даже с публичным anon-ключом.
- **Имена** (`booking_names`) и **визиты** (`visits`) читает только
  аутентифицированный администратор. Снятие брони — тоже только админ.
- Вход в админку — **Supabase Auth** (email + пароль). Публичная регистрация
  отключена. Аккаунт админа создан через service_role.

Полная схема и политики — в `data/schema.sql`. Если `js/config.js` не заполнен,
сайт откатывается на демо-режим localStorage (бронь только в одном браузере).

## Панель администратора
- Адрес: `/admin.html` (закрыт от индексации `noindex`).
- Вход по email/паролю Supabase Auth.
- Показывает: уникальных гостей, всего заходов, последний заход, таблицу
  броней (позиция / кто / когда) и журнал визитов (время / устройство).
